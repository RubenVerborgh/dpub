EPUB 3

EPUB 3 is a very mature specification

  • Widely used format for digital publication
  • The only truly portable format (and not bound to a particular environment)
  • Can be used for books, magazines, journal articles, documentation…

What is the secret?

Well, at least one of the secrets…

  • EPUB is based on Web Technologies like HTML, CSS, SVG, etc.
    • collectively referred to as “Open Web Platform” (OWP)
  • I.e., no rendering, formatting, etc, specification had to be defined
  • Instead, EPUB relies on specifications defined by W3C and deployed, e.g., in browsers

Bottom line: relationships of EPUB with OWP is vital

Are we all done? Ie, is OWP perfect for EPUB?

There are two major areas that need work:

  1. bring OWP and Digital Publishing even closer
  2. bring the Web and Digital Publishing even closer

Bring OWP and Digital Publishing even closer

Missing OWP features

  • Some features, needed by publishing, are missing from OWP; examples:
    • pagination, page controls for headings, page numbers
    • graphics transition effects (needed in, e.g., mangas, cartoons, children books)
    • accessibility oriented structuring of complex documents
    • dynamic numbering of pages, index references, etc, over several (HTML) files
  • W3C has a Digital Publishing Interest Group to look into these issues and cooperate with the relevant W3C groups

Bring OWP features to publishing

  • For example:
    • dynamic tools, deployed on the Web, and would be beneficial in publishing
    • educational publications reach out to external services on the Web via extra protocols
    • new authentication methods could change the way publications are accessed
    • etc.

Bring the Web and Digital Publishing even closer

What is, in fact, a (digital) book?

A book is a discrete collection of text (and other media), that is designed by an author(s) as an internally complete representation of an idea, or set of ideas; emotion or set of emotions; and transmitted to readers in various formats.

“What books can learn from the Web / What the Web can learn from books”, from Hugh McGuire, Medium

  • This is independent of format: the same “book” should be available in different format and media
  • Instead: a Digital Book is different than a Web page (conceptually and practically)
  • This should not be the case!

What does this mean?

Separation between publishing “online”, as Web sites, and offline and/or packaged should be diminished to zero

  • This means:
    • publication content on the Web can be loaded into a browser or a specialized reader, whatever the user prefers
    • a publication on a local disc can be pushed onto the Web and used without any change
    • content are authored regardless of where they are used
    • these are done without any user interaction, possibly automatically
